# Testo del compito
I punteggi riportati sono quelli relativi alla valutazione delle **capacità di uso base** nell'uso degli strumenti di 
versionamento (Git, GitHub e SourceGit)

## Parte 0: Ben arrivato [2 punti]
Se stai **leggendo questa parte e se arrivato qui da solo**, hai già accumulato **[2 punti]**, ottimo lavoro!

## Parte 1: Uso di Git e GitHub
Le operazione che seguono devono essere eseguite seguendo l'ordine in cui sono presentate.

### Task 1.1: Creazione del proprio branch [1 punto]
Creazione del branch usando il nome studenti/cognome dove è il cognome dello studente in minuscolo

### Task 1.2: Push del proprio branch sul repository [1 punto]
Assicurarsi di essere collaboratori del progetto inviando all'insegnante (via Teams o a voce) il proprio username di 
GitHub. Una volta ottenuto lo status di collaboratore, procedere con il push del proprio branch sul repository.

### Task 1.3: Implementare il codice [2 punti]
Implementare in locale il codice come indicato dal file CODICE.md, effettuando frequenti commit come da "best practice" 
di git.
In particolare, è *opportuno* fare un commit:
 - Ogni volta che si risolve un problema al codice
 - Ogni volta si implementa una nuova funzione/funzionalità
 - Ogni volta si aggiunge un nuovo file
#### EVITARE LE PUSH
Visto che lavorate senza Internet è sbagliato provare a fare le PUSH a ogni modifica (cosa che normalmente con internet
disponibile sarebbe, molto conveniente e caldamente consigliata) 

### Task 1.4: Push finale [1 punto]
Effettuare il push finale del proprio branch sul repository

### Task 1.5: Merge dei risultati [2 punti]
Effettuare il merge del proprio branch nel branch principale del repository, quello da cui abbiamo creato 
il proprio branch inizialmente.
